# 1. React Context

Вы уже познакомились в React с 

- state, props
- hooks + life cycle
- refs

На их основе уже можно строить любую логику на React!
Но с ростом приложения довольно быстро появляются некоторые неудобства.

Главные из них:

- Прокидывание props (data), callbacks (setters) вниз по дереву компонентов  с самого верхнего уровня.
- Рендеринг и логика мешаются в одном месте. Это быстро раздувает компоненты и усложняют их понимание.
- Добавляются неконтролируемые побочные эффекты вперемешку с обновлением данных.
